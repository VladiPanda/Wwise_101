ATRAC9 decoding failed.

ATRAC9 하드웨어 디코더가 오디오 스트림 디코딩을 시작하거나 지속하지 못했습니다. 사운드가 중단되었습니다.

유력한 원인:

  • 작업 버퍼를 초기화하기에 메모리가 충분하지 않습니다. 이 경우 메모리 부족 오류가 발생하게 됩니다.

  • 사운드에 Zero Latency 옵션이 활성화된 경우 Zero Latency 버퍼링이 소비되고 나면 미디어에서 SoundBank 안에 있는 부분이 스트림되는 WEM 파일과 대응하지 않게 됩니다. 이 현상은 다음과 같은 원인으로 일어날 수 있습니다.

    • 게임에서 뱅크와 WEM의 패키징 혹은 배포 오류. 파일 중 하나라도 복사되지 않으면 이 문제가 일어나게 됩니다.

    • 사운드가 두 개의 뱅크에 복제되어 Zero Latency의 양도 함께 복제되었습니다. 한 뱅크만 재생성될 경우 이 문제가 일어날 수 있습니다.

    • 오래된 뱅크가 게임에 로딩되었습니다.

  • 이 스트림을 읽을 때 I/O 연결에서 문제가 생겼습니다. 이 문제는 보통 맞춤화된 I/O 연결 사용과 관련됩니다.

권장 해결 단계:

  • 뱅크를 재생성하세요.

  • 오디오 파일 캐시를 제거하고 WEM 파일을 다시 변환하세요. 캐시 파일 제거는 File 메뉴에서 실행하거나 .cache 폴더에서 특정 WEM 파일을 직접 삭제함으로써 실행할 수 있습니다.

  • AK::SoundEngine::Init을 호출하기 전에 AkMemSettings.uMemAllocationSizeLimit를 변경하여 메모리 제한을 늘리세요. 초기화 설정에 대한 더 많은 정보는 고급 사운드 엔진 통합을 봐주세요.
