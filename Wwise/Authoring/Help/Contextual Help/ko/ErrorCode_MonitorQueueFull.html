Monitor Queue full. Increase the size of uMonitorQueuePoolSize. Message size ... bytes.

Wwise Profiler에서 사용하는 Monitor Queue가 꽉 차서 한 메시지를 포함할 수 없었습니다. 이 오류의 유일한 부작용은 Wwise Profiler에서 메시지가 누락된다는 것입니다.

유력한 원인:

  • API 프로파일링을 사용할 때 매우 짧은 기간에 API 호출이 급증하면 이 문제가 발생할 수 있습니다.

  • 동일한 CPU 코어의 스레드 우선 순위로 인해 모니터링 스레드를 제 시간에 실행할 수 없습니다.

권장 해결 단계:

  • AkInitSettings::uMonitorQueuePoolSize를 늘리세요.

  • API 호출 급증을 방지하도록 AK::SoundEngine::RenderAudio를 더 정기적으로 호출하세요.

  • CPU 프로파일러를 사용하여 동일한 CPU 코어를 공유하는 모든 스레드의 스레드 우선 순위를 확인하세요.
