Duplicated media has different length in bank X. Check previously unloaded bank. Stopping sound.

이 오류는 다음 기준이 모두 부합할 경우 일어납니다.

  • 미디어가 두 개 이상의 뱅크에 복제되었습니다. 이 현상은 한 사운드가 두 뱅크에 추가되거나, 두 개의 다른 사운드가 서로 다른 두 뱅크에서 동일한 미디어 파일을 사용할 경우에 일어납니다.

  • 한 뱅크에 Prefetch 섹션만 있습니다 (Zero Latency Streaming의 경우)

  • 다른 뱅크에 더 긴 Prefetch 섹션이 있거나 (Zero Latency Streaming의 경우) 혹은 전체 미디어가 있습니다.

  • 이 미디어를 사용하는 재생 중인 사운드가 있습니다.

  • 해당 미디어가 들어 있는 뱅크가 언로드되었습니다.

위의 모든 조건이 성사할 경우, 해당 메시지가 'Bank Unloaded' 메시지 바로 다음에 표시됩니다. 이 메시지는 관련된 한 뱅크의 이름을 표시하며, Bank Unloaded 메시지는 다른 뱅크의 이름을 참조합니다. 또한 Capture Log의 Wwise Object 열에서 이 미디어의 복사본을 재생하고 있었던 Sound를 표시해줍니다. 해당 Sound는 즉시 중단됩니다.

유력한 원인:

  • 최신 데이터를 가지지 않은 오래된 뱅크 (오래된 빌드에서) 가 로드되었습니다.

  • Wwise Project에 서로 다른 Prefetch 길이를 가진 두 개의 사운드가 서로 다른 두 뱅크에 각각 들어 있습니다.

권장 해결 단계:

  • Wwise Project의 .cache 폴더를 비우고, SoundBank를 재생성하고, 다시 패키징하고 (필요한 경우), 대상 콘솔로 재배포하세요.

  • Wwise Project에서 해당 오류가 참조하는 Sound를 찾으세요. 동일한 미디어 파일을 사용하는 다른 Sound를 찾으세요. 이 사운드들이 모든 경우에서 동일한 Zero Latency 길이를 사용하는지 확인하세요.
