File header too large due to markers or envelope.

The file being played has failed starting because parsing the markers or the volume envelope data (for HDR processing) requires too much memory or exceeds the file format header constraints. The file cannot be read.

Probable causes:

  • Automatic marking for transients was enabled in the source options. If the threshold was too fine it may generate too many markers.

  • Envelope generation was done with a fine threshold, too many points were generated.

  • The original file had too many markers.

  • The original file had big data markers. This is common with files that have animation data for lip sync.

Recommended resolution steps:

  • Change the automatic marking options in the Source Editor in Wwise. Then regenerate and redeploy your SoundBanks.

  • Remove unused markers from the source file.

  • Change the automatic envelope analysis options in the Source Editor in Wwise. Then regenerate and redeploy your SoundBanks.
