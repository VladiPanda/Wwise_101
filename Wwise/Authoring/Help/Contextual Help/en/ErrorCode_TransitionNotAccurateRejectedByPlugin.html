Transition not sample-accurate due to incompatible encoding parameters.

This error occurs when two sounds with the same audio format but different encoding parameters are set up to play one after the other in a sample-accurate transition. Some hardware-accelerated codecs require sounds to have identical encoding parameters in order to perform sample-accurate transitions. Such transitions can exist in Random/Sequence Containers, in Interactive Music, and when using the Dynamic Sequence mechanism in code.

Recommended resolution step:

  • Identify the sounds involved and change their Conversion Settings so all encoding parameters match.
