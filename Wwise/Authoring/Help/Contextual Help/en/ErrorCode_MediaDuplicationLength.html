Duplicated media has different length in bank X. Check previously unloaded bank. Stopping sound.

This error occurs if all following criteria are true:

  • Media is duplicated in two or more banks. This will happen if one sound is added to two banks, or if two different sounds use the same media file, in two different banks.

  • One bank has only a Prefetch section (for Zero Latency Streaming)

  • The other bank has a longer Prefetch section (for Zero Latency Streaming) or the full media.

  • There is one actively playing sound using this media.

  • One bank containing said media is unloaded.

When all the above conditions are met, the message will appear immediately after the "Bank Unloaded" message. This message will contain the name of one of the banks involved, and the Bank Unloaded message will refer to the other bank. Also, the Wwise Object column of the Capture Log will show the Sound that was playing using these copies of media. The Sound will be stopped immediately.

Probable causes:

  • Some stale banks (from older builds) were loaded, with out of date data.

  • The Wwise Project has two separate sounds with different Prefetch length, in two separate banks.

Recommended resolution steps:

  • Clear the .cache folder of the Wwise Project, regenerate your SoundBanks, repackage them (if needed) and redeploy to the target console.

  • Find the Sound referred by the error message in your Wwise Project. Find other Sounds using the same media file. Ensure they all use the same Zero Latency length in all cases.
