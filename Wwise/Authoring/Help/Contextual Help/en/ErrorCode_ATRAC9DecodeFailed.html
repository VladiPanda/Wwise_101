ATRAC9 decoding failed.

The ATRAC9 hardware decoder could not start or continue decoding an audio stream. The sound was stopped.

Probable causes:

  • There was not enough memory to initialize working buffers. This would be preceded by an out of memory error.

  • If the sound has the Zero Latency option enabled, the media portion that is in the SoundBank doesn't match the WEM file that is streamed, once the Zero Latency buffering is consumed. This can happen because of:

    • An error in the packaging or deployment of the bank and WEM in the game. Failure to copy one of the file would result in this problem.

    • The sound is duplicated in two banks, thus duplicating the Zero Latency portion. If only one bank is regenerated, this problem can occur.

    • An old bank is loaded in game.

  • There was a problem in the I/O hook while reading this stream. Usually related to the use of a custom I/O hook.

Recommended resolution steps:

  • Regenerate your banks.

  • Clear the audio file cache and reconvert the WEM files. You can clear the file cache through the File menu, or by manually deleting the specific WEM file in the .cache folder.

  • Increase the memory limit by changing AkMemSettings.uMemAllocationSizeLimit before calling AK::SoundEngine::Init. See Advanced Sound Engine Integration for more information on initialization settings.
