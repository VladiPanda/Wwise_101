Type mismatch while loading bank. Object ... is a ... in the currently loading bank. It was a ... in bank ...

This error indicates a collision in the unique identifiers used in Wwise. Two objects share the same identifier in two different SoundBanks.

Possible causes:

  • Usually, there is an out-of-date, unused SoundBank on the console test directory.

  • There is a genuine identifier collision, however, this would be extremely rare.

Recommended resolution steps:

  • Delete all .bnk files from the test console.

  • Change the name of the reported object in the Wwise Project, and regenerate the SoundBanks.

  • Contact Audiokinetic Support since this could also be an internal bug.
