AddOutput() needs unique Listeners for multi-instance outputs using the same Audio Device type.

When using the same Audio Device type for multiple outputs, such as is the case for Game Controller audio or motion, sound routing can not be resolved only with the Audio Device type. You must specify unique listeners for each of the instances created with the same Audio Device type. This is necessary to get a different audio mix for each output.

Recommended resolution steps:

  • Register a new Game Object to use as listener for each output using the same type.

See also:

  • Integrating Secondary Outputs

  • Integrating Wwise Motion, section Multiplayer considerations.

  • The sample IntegrationDemoMotion in SDK\samples\IntegrationDemo demonstrates this situation with both motion and audio outputs.
